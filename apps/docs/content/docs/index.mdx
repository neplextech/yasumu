# Welcome to Yasumu

**Yasumu** is a modern, free, and open-source API laboratory for
designing, testing, and debugging API workflows. Built for developer
productivity, speed, and freedom — without vendor lock-in.

<Callout type="info" title="Beta Software">
  Yasumu is actively under development. You may encounter unfinished
  features, bugs, or inconsistent behavior as we rapidly prototype and
  iterate. We welcome your feedback and contributions!
</Callout>

## Why Yasumu?

Unlike traditional API clients that store data in hidden databases or
require cloud accounts, Yasumu operates like an **IDE for your APIs**.
Your configurations live directly in your project folder, making
collaboration and version control seamless.

It feels like the tools you already know, but with the flexibility to
handle complex, experimental workflows directly in your file system.

### What Makes It Different

Everything lives in your project folder as plain `.ysl` files. Commit
them, diff them, review them in PRs—your API collections are just
another part of your codebase.

The desktop app runs on Tauri and uses the embedded Deno-based Tanxium
adapter for scripting. Request construction, execution, interpolation,
tests, and workspace behavior live in a shared headless engine. The
Node.js CLI assembles that same engine, so `.ysl` workflows behave the
same way in the editor, terminal, and CI.

No account required. No cloud sync. No paid tiers.

## Concepts & Workflow

Yasumu acts as an **IDE for your API interactions**. Unlike tools that
hide data in proprietary databases or cloud silos, Yasumu lives in
your repo.

### Project-Based Workspaces

When you initialize Yasumu in a project, it creates a `yasumu`
directory. Your definitions, tests, and workflows are stored here
using the **Yasumu Schema Language (.ysl)** — a custom format designed
for readability and clean git diffs.

### Collaboration via Git

- **Commit & Push:** Treat your API collections as code.
- **Review:** `.ysl` files are text-based, making Pull Request reviews
  easy.
- **Sync:** Teammates simply pull the branch to get the latest API
  workflows.
- **Secure:** Sensitive secrets are managed via local environment
  variables, never committed to version control.

### Runtime Architecture

Tauri provides the desktop shell and OS-level operations. A
runtime-independent headless package owns request and workspace
behavior. Tanxium implements the script runtime contract for the
desktop, while a Node.js worker adapter implements it for the CLI.
Both consume one generated script API and virtual-module contract.

## Tech Stack

- **Frontend:** Next.js with React
- **Desktop Shell:** Tauri
- **Headless Engine:** Shared TypeScript domain and execution services
- **Script Runtimes:** Embedded Tanxium (desktop) and Node.js workers
  (CLI)
- **Platforms:** Windows, Linux, macOS

## Project Philosophy

- **Open Source by Default** — Transparency builds trust. The entire
  codebase is available on GitHub.
- **Zero Vendor Lock-in** — Your data belongs to you, stored in
  readable files on your disk.
- **Developer Experience First** — Fast, beautiful, and
  keyboard-friendly.
- **No Artificial Paywalls** — All features are free to use.
- **Sustainability** — Built for long-term maintainability over hype.
